# CRUD 分类审批模式技能

将 OpenClaw 的操作行为按 CRUD（创建/读取/更新/删除）进行分类管理，实现精细化的审批控制。

## 核心设计理念

- **查询(Read)**：完全放开，提升效率
- **创建(Create) + 更新(Update) + 编辑(Edit) + 删除(Delete)**：增加隔离层，返回行为清单进行二次确认

## 分类定义

| 操作类型 | 审批要求 | 说明 |
|---------|---------|------|
| **Read** | ❌ 无需审批 | 文件读取、搜索查询、目录浏览等 |
| **Create** | ✅ 需二次确认 | 文件创建、新技能创建、目录创建等 |
| **Update/Edit** | ✅ 需二次确认 | 文件修改、代码编辑、配置更新等 |
| **Delete** | ✅ 需二次确认 | 文件删除、目录删除、技能删除等 |

## 工作流程

### 1. 无审批流程（Read）
```
用户请求 → 直接执行 → 返回结果
```

### 2. 审批流程（Create/Update/Edit/Delete）
```
用户请求 → 分析操作 → 生成行为清单 → 用户确认 → 执行操作 → 返回结果
              ↓
         显示：操作类型、影响文件、变更预览
```

## 行为清单格式

当需要确认时，应返回如下格式的清单：

```
⚠️ **操作确认** - [操作类型]

**影响范围：**
- 文件1: path/to/file1
- 文件2: path/to/file2

**操作说明：**
[详细描述将要执行的操作]

请回复以下任一指令继续：
- ✅ **确认执行** - 继续执行此操作
- ❌ **取消** - 取消此操作
- 🔄 **修改** - 修改操作后重新确认
```

## 使用场景

- 日常开发：快速查询，不受审批限制
- 关键操作：所有写操作前有确认机制
- 技能开发：新技能创建需要确认，确保安全

## 配置

此技能不需要额外配置文件，会自动注入到会话中使用。
