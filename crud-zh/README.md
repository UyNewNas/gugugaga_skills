
# 🦞 CRUD 分类审批模式 - OpenClaw 技能

将 OpenClaw 的操作行为按 CRUD（创建/读取/更新/删除）进行分类管理，既保证效率，又确保安全。

## 🎯 设计理念

**痛点：**
- **效率优先：查询完全放开，不卡流程
- **安全兜底：创建、更新、编辑、删除增加二次确认

## 📋 分类规则

| 操作类型 | 审批要求 | 说明 |
|---------|---------|------|
| **Read（读取）** | ❌ 无需审批 | 文件读取、搜索查询等 |
| **Create（创建）** | ✅ 需二次确认 | 文件创建、新技能创建等 |
| **Update/Edit** | ✅ 需二次确认 | 文件修改、代码编辑等 |
| **Delete（删除）** | ✅ 需二次确认 | 文件删除、目录删除等 |

## 🚀 快速开始

### 安装

```bash
# 复制到 OpenClaw 技能目录
cp -r crud ~/.openclaw/skills/crud

# 重启 Gateway
openclaw restart
```

### 使用

安装后自动生效，无需额外配置。

## 📁 目录结构

```
crud/
├── SKILL.md      # 技能定义
├── CLAUDE.md      # 核心指令（自动注入）
├── README.md     # 本文件
├── INSTALL.md    # 安装指南
└── EXAMPLES.md   # 使用示例
```

## 🔄 工作流程

### 无需审批（Read）
```
用户请求 → 直接执行 → 返回结果
```

### 需要审批（Create/Update/Edit/Delete）
```
用户请求 → 分析操作 → 生成行为清单 → 用户确认 → 执行操作 → 返回结果
```

## 📝 行为清单示例

```
⚠️ **操作确认请求**

**操作类型：** 更新/编辑
**时间戳：** 2026-03-23 10:30:00

---

**📄 影响文件清单：**
1. `main.py` - 修改端口配置

---

**📝 操作详情：**
- 将 main.py 中的端口号从当前值修改为 8080

---

**请选择：**
- ✅ **确认执行** - 继续执行上述操作
- ❌ **取消** - 放弃本次操作
- 🔄 **修改要求** - 调整操作后重新确认
```
