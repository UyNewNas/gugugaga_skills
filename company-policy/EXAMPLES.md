# Company Policy Query - Usage Examples

## Example 1: Check-in Questions

### User Input
```
What are the check-in requirements?
```

### Expected Output
```
📋 Check-in Requirements

## Multi-system Parallel Usage

You need to use multiple systems for check-in:

| System | Purpose | Importance |
|--------|---------|------------|
| **elink** | Daily check-in | ⭐ **Only basis for settlement** |
| **Face Recognition** | Door access | Will be included in settlement |
| **DingTalk** | Daily check-in | Auxiliary record only |
| **Tianyang APP** | Daily check-in | Internal record |

## Time Norms

- System default shows 9:00-17:00 (not actual attendance time)
- Follow project team requirements for actual check-in times
- Late/early departure (even 1 minute) is a violation

## Location Norms

- ❌ Strictly prohibited to check in on the road
- ✅ Must report location anomalies immediately
- ✅ For going out, apply via elink first

## Photo Evidence (Important!)

Take photo immediately after each face recognition:
- Earliest clock-in of the day
- Latest clock-out of the day
- Ensure photo clearly shows time

💡 Photos serve as appeal evidence if attendance data is abnormal!
```

## Example 2: Work Hours Reporting

### User Input
```
How do I fill in work hours correctly?
```

### Expected Output
```
📋 Work Hours Reporting Guidelines

## Basic Requirements

1. **When to fill**: After clock-out each day
2. **Format**: Morning and afternoon sections
3. **Length**: 15-100 characters per entry
4. **Content**: Detailed and quantified

## Content Guidelines

✅ **Do**:
- Support daily workload with specific details
- Use "配合" (assist) or "协助" (support) instead of "负责" (responsible)
- Show quantified indicators for long-term items

❌ **Don't**:
- Write identical content for a whole week
- Include sensitive information (procurement, project initiation, feasibility study, party building, recruitment)
- Leave blank on work days (except approved leave)

## Important Notes

| Situation | Action |
|-----------|--------|
| Leave/Vacation | Don't fill work log (leave blank) |
| Forgot check-in | Upload supplementary check-in approval + work evidence |
| Half-day leave | Note morning/afternoon leave in reason |

## Deadlines

- **Weekly**: Friday 10:00 PM (can modify within the week)
- **Monthly**: 15th of following month (locked)

⚠️ Work hours not filled = No settlement for that day!
```

## Example 3: Leave Process

### User Input
```
I need to take leave tomorrow, what should I do?
```

### Expected Output
```
📋 Leave Application Process

## Three Required Steps

### Step 1: Get CSG Project Leader Approval
- Contact China Southern Power Grid project leader
- Get verbal or written agreement
- This is the prerequisite for leave application

### Step 2: Submit in elink
- Log in to elink
- Fill leave application
- Include: reason, time, duration
- Submit for approval

### Step 3: Submit in Tianyang APP
- Open Tianyang APP
- Navigate: Workbench → Attendance Leave → Apply for Leave
- Fill same details as elink
- Submit for approval

## Important Notes

⚠️ **Must apply in advance** - Cannot be retroactive!

⚠️ **After approval** - Cannot withdraw or cancel

⚠️ **No work hours needed** - Don't fill work log on leave day

⚠️ **Sync to CSG** - Must notify CSG project manager via elink or email

## Half-day Leave Special Notes

If taking half-day leave:
- ✅ Must complete clock-in AND clock-out in elink and Tianyang APP
- ✅ If afternoon leave: clock out when leaving at noon
- ✅ Forgot check-in or system failure: must supplement

## Extended Leave (3+ days)

Additional requirement:
- Must sync to CSG department head via elink or email

---

💡 Need help? Contact Assistant He Ziban (何紫班) via WeChat or elink
```

## Example 4: Overtime Process

### User Input
```
How does overtime work?
```

### Expected Output
```
📋 Overtime Process

## Two Required Steps (Both Must Be Completed)

### Step 1: Overtime Application
- **Who submits**: China Southern Power Grid department
- **When**: Before overtime starts
- **Requirement**: Approval must be completed before overtime begins

### Step 2: Check-in
- **Where**: elink
- **When**: According to actual time
- **Note**: Supplementary check-in is INVALID for overtime

## Overtime Calculation

| Type | Start Time | Notes |
|------|------------|-------|
| Weekday | 19:00 | Minimum 1 hour unit |
| Weekend | As applied | Application must complete before start |
| Legal holiday | As applied | + Email/elink to department head |

## Deductions

- 12:00-13:30 (lunch break)
- 17:30-19:00 (dinner break)

## Important Rules

✅ Must have clock-in AND clock-out times
✅ No need to fill separate work log for overtime
❌ Supplementary check-in is invalid for overtime
❌ Don't clock out early (invalidates overtime)

## Morning Leave + Evening Overtime

If you took morning leave and want overtime to count:
- Must clock in via elink and door before noon (before 13:30)

## Overtime Data Missing?

1. Check if both steps completed
2. Contact CSG department assistant (not Tianyang personnel)
3. Resolve before final settlement sheet is issued

---

💡 Overtime pay is based on CSG final settlement sheet only
```

## Example 5: Supplementary Check-in

### User Input
```
I forgot to check in this morning, what should I do?
```

### Expected Output
```
📋 Supplementary Check-in Process

## Limits & Deadlines

| System | Limit | Deadline |
|--------|-------|----------|
| **elink** | 6 times/month | 10th of following month |
| **Tianyang APP** | No monthly limit | 3 working days |

⚠️ Exceed limits → Appeal process (only 3 times/year!)

## Steps

### 1. Fill Actual Time
- Clock-in: before 8:30
- Clock-out: after 17:30
- ❌ Don't use system default time

### 2. Select Witness
- Choose colleague from same team
- ❌ Cannot select yourself

### 3. Upload Evidence

| Situation | Evidence Required |
|-----------|-------------------|
| Account not given | Door photo (with date, time, face) OR DingTalk record |
| Normal forgot check-in | elink work discussion screenshot OR DingTalk record |

### 4. Submit for Approval
- Approver: Tianyang project manager

## Anomaly Handling (Location Drift, System Bug, Door Malfunction)

⏰ **Within 10 minutes**, collect 4 proofs:
1. DingTalk check-in
2. Tianyang APP check-in
3. Door photo
4. elink work communication screenshot

📤 Send all proofs to Tianyang project assistant for filing

📝 Naming format: Name + Date + Anomaly Type
Example: "张三0529定位漂移"

---

💡 Work duration must be 8 hours, otherwise calculated as half day!
```

## Example 6: Information Security

### User Input
```
What are the network security rules?
```

### Expected Output
```
📋 Network Security Regulations

## Terminal & Software Security

✅ **Do**:
- Use strong passwords
- Follow company terminal management norms

❌ **Don't**:
- Use empty or weak passwords
- Install unauthorized software (QQ, Weibo, Douyin, etc.)
- Close or uninstall antivirus/anti-leakage software

## Account & Password Protection

✅ **Do**:
- Lock screen when leaving workstation
- Encrypt important files
- Clear browser history regularly

❌ **Don't**:
- Post passwords or IP addresses in exposed areas
- Save passwords in browser

## Network Behavior

❌ **Strictly Prohibited**:
- Publishing company info to internet
- Setting up wireless hotspots
- Bypassing security controls
- "One machine dual network" (一机双网)

## elink Usage

✅ **File Transfer**:
- Encrypt important files
- Use certified hard drive for files >500MB

❌ **Content**:
- No negative, attacking, political, or sensitive speech
- Screenshots and complaints = serious violation

❌ **Login**:
- No auto-login
- Must log out when leaving

## Emergency Response

🚨 If you suspect network security incident:
1. Report immediately
2. Preserve the scene
3. Cooperate with investigation

---

⚠️ Violations may result in contract fines and legal consequences!
```
